Key Design Decisions
Concern Recommendation
Chunk granularityOne chunk per table (name + columns + FKs + enums). Never split a table's columns across chunks — a partial schema is worse than no schema.
Retrieval kStart at 8–10 tables; tune down guided by schema_recall scores from the eval pipeline.
FK stitchingAfter retrieval, auto-include any table referenced by a FK in a retrieved chunk to prevent broken join chains.
Embedding modelUse the identical model for ingestion and query-time. A mismatch silently degrades recall with no visible error.
Prompt cachingCache the retrieved schema slice (not a full 50-table dump) — keeps Anthropic prompt caching effective and cheap at scale.
Re-ingestion triggerRun the ingestion pipeline on every schema migration. Wire it into CI alongside your migration tool (Alembic, Flyway, etc.).
